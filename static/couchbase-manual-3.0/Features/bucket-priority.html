Disk I/O priority

Disk I/O priority allows workload priorities to be set at the bucket level.

Bucket priority settings can be specified at the bucket-level. The bucket disk I/O priority can be set as either high or low, whereas, low is the default. Bucket priority settings determine the number of threads assigned to a bucket allocated resources across buckets. Bucket latency and I/O operations are impacted by the setting value. When a bucket has a high priority, additional I/O resources are assigned for processing tasks.

The default buckets settings can be set during initial setup.

Note: Buckets normally do not have less than three (3) threads.

Backward compatibility

When upgrading from a 2.x release to a 3.x release, Couchbase converts an existing thread value to either a low or a high priority based on the following:

  • Buckets allocated six to eight (6-8) threads are high priority.
  • Buckets allocated three to five (3-5) threads are low priority.
  • Setting disk I/O priority (UI)
    The disk I/O priority for a bucket is set via the Data Bucket panel either when creating a data bucket or when editing a data bucket.
  • Setting disk I/O priority (CLI)
    To set the disk I/O priority for a bucket, use the couchbase-cli tool bucket-create or bucket-edit command and the --bucket-priority=[low|high] option.
  • Setting disk I/O priority (REST API)
    The disk I/O priority for a bucket is set with the /pools/default/buckets/bucket_name URI and the maxNumberThreads option.
