Couchbase 3.0 features

The Couchbase Server 3.0 release includes the following new features:

  • Database Change Protocol
    Database Change Protocol (DCP) is the protocol used to stream data changes to buckets.
  • Shared thread pool
    A shared thread pool is a collection of threads which are shared across multiple buckets.
  • Tunable memory
    Tunable memory allows both value-only ejection and full metadata ejection from memory.
  • Disk I/O priority
    Disk I/O priority allows workload priorities to be set at the bucket level.
  • Graceful failover
    Graceful failover allows administrators to safely failover a node from the cluster after all in-flight operations are completed and without data loss.
  • Delta node recovery
    Delta node recovery allows nodes to be re-added to the cluster and incrementally caught up with the data changes.
  • Incremental backup and restore
    Incremental backup and restore enables administrators to quickly back up only modified data in the database, making backup and restore more efficient for larger data sets. Enterprise Edition only.
  • Stream-based views
    Stream-based views reduce latency for view updates, providing faster view consistency and fresher data.
  • XDCR pause and resume replication
    During XDCR replication, the process can be paused and resumed.
  • Stream-based XDCR
    Stream-based XDCR collects data changes from memory on the source cluster and streams the data changes directly to memory on the destination cluster.
  • Encrypted admin access
    Encrypted administrator access provides access using Secure Socket Layer (SSL) authentication. Enterprise Edition only.
  • Encrypted data access
    The Couchbase Server client libraries support client-side encryption using the Secure Sockets Layer (SSL) protocol. Enterprise Edition only.
  • Cluster-wide diagnostics
    The Couchbase support team uses logs and other system diagnostics to analyze customer issues.
