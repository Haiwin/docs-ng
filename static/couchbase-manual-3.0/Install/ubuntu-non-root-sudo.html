    Installing as non-root, non-sudo

    Installing on Ubuntu as non-root, non-sudo is for development purposes only.

    There may be cases when you want to install the server as a non-root, non-sudo user. If you perform a non-sudo, non-root installation you will be still be able to run Couchbase Server and all Couchbase command-line tools. To do so on Ubuntu:

    1. After downloading the Couchbase DEB package, go to the directory where it is located and extract it:

      > dpkg-deb -x couchbase-server-community_x86_64_2.0.0-1767-rel.deb $HOME

      In the directory where you extracted the files, you will see opt and etc subdirectories.

    2. After you extract the Couchbase Server install files, go to the subdirectory:

      cd opt/couchbase

    3. Run this password-related script:

      ./bin/install/reloc.sh `pwd`

      This enables you to continue the install as a non-root, non-sudo user.

    4. To run the server:

      ./bin/couchbase-server -- -noinput -detached

    5. To stop the server:

      ./bin/couchbase-server -k
