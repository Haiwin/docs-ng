    Release notes

    This release note is for the 3.0.0 release. It contains a list of known and fixed issues.

    Couchbase Server is a NoSQL document database for interactive web applications. It has a flexible data model, is easily scalable, provides consistent high performance and is “always-on,” meaning it can serve application data 24 hours, 7 days a week.

    The new major features in this release include:

    • Performance
      • Tunable memory (option to remove keys and metadata from memory).
      • Bucket disk I/O ejections (for faster persistence to disk).
      • Faster warm up from disk when restarting the server.
    • Availability and reliability
      • Faster XDCR without persisting to disk on the source cluster.
      • Index building without persisting to disk on the source cluster.
    • Ease of administration
      • Incremental backup
      • Incremental restore
    • Security
      • Data and password encryption between client and server
      • Secure administrator access

    New features and behavior changes

    The following major enhancements include changes to the infrastructure, memory managment, server maintenance, views indexing, and XDCR.

    • Infrastructure enhancement with Database Change Protocol (DCP) and Shared thread pool

      For DCP statistics via the Web console and CLI, see DCP Queues via UI and DCP stats via CLI.

    • Bucket and disk I/O enhancements including:
      • Bucket metadata ejections - For more information about bucket metadata ejections, see Managing metadata in memory, Setting metadata ejection policy, and Setting metadata ejection
      • Disk I/O priority - see Managing disk I/O priority, Setting bucket priority, and Setting disk I/O priority
    • Server maintenance enhancements including:
      • Graceful failover - For more information about graceful failover via the web console, CLI, and REST API, see Failing over a node, Failing over nodes, and Setting graceful failover
      • Delta node recovery - For information about using delta node recovery via the web console, CLI, and REST API, see Recovering a node, Recovering nodes, and Setting recovery type.
      • Incremental backup and restore - For information about using incremental backup and restore via the CLI, see Backing up incrementally and Restoring incrementally.
    • Index building enhancement with Stream-based views.
    • XDCR enhancements with Stream-based XDCR and XDCR pause and resume replication - For more information about pausing and resuming XDCR replication, see Pausing XDCR replication, Pausing XDCR replication streams, and Pausing XDCR replication streams.
    • Security enhancement including Encrypted admin access and Encrypted data access
    • Log and log collection improvements with Cluster-wide diagnostics - For more information about collecting diagnostics, see Managing diagnostics, Diagnostics with couchbase-cli, and Diagnostics with cbcollect_info,

    Couchbase .NET incompatibility

    An incompatibility between Microsoft Framework 4.0 or earlier, the Couchbase .NET SDK 1.3.7 or earlier, and Couchbase Server 3.0 has been identified where the URL generated for a View operation on a Bucket is improperly encoded. This causes view requests to fail with the following message in the body:

    "Design document not found, body: {"error":"not_found","reason":"missing"}"
    Impact
    The affected Microsoft .NET versions are 4.0 or earlier, Couchbase .NET SDK 1.3.7 or earlier, and Couchbase Server 3.0 (including Beta) >= 3.0. Microsoft .NET version 4.5 changes the way Uri.ToString handles it’s Unicode encoding, which resolves the issue, thus clients running on that version or higher of the CLR are not impacted. Additionally, Couchbase Server 2.5 and lower are not impacted.
    Note: Couchbase .NET 1.3.8 has a patch which resolves the issue for all versions of Couchbase Server and Microsoft .NET Frameworks.
    Workaround
    In addition to the patch released with Couchbase .NET SDK version 1.3.8, add the following element to your App.Config or Web.Config to enable support for all SDK and Server versions. 
    <uri>
    <iriParsing enabled="true"/>
<uri>

    Customers who are not ready or cannot upgrade their Couchbase .NET SDK’s to 1.3.8 or greater are strongly advised to aentry this element to their deployments if they intend to use the Couchbase .NET SDK with Couchbase Server 3.0 or greater.

    References
    https://connect.microsoft.com/VisualStudio/feedback/details/758479/system-uri-tostring-behaviour-change

    Views behavior change

    The stale=false view query argument has been enhanced. It considers all document changes which have been received at the time the query was received. This means that using the durability requirements or observe feature to block for persistence in application code before issuing the falsestale query is no longer needed. It is recommended that you remove all such application level checks after completing the upgrade to the 3.0 release.

    A bad replicas error message means that replication streams could not be created. There may be many reasons for this to happen, however, if some of the vBuckets sequence numbers (which are maintained internally in Couchbase) are invalid, the a log message displays in the memcached logs similar to the following:

    (DCP Producer) some_dcp_stream_name (vb 0) Stream request failed because the snap start seqno (100) <= start seqno (101) <= snap end seqno (100) is required

    In order for a DCP producer to accept a request for a DCP stream, the following must be true:

    snapshot start seqno <= start seqno <= snapshot end seqno

    If the above condition is not true for a stream request, contact support to resolve the issue.
