6.4. サーバーノード

クラスタ内のすべてのノード上のバケットの監視に加えて、Couchbase Serverは、個々のノードの統計情報の監視もサポートしています。

サーバーノードの監視概要は、スワップ使用量、RAM使用量、CPU使用率と、クラスタ内のすべてのノードでアクティブな項目のサマリーデータを示しています。

図6.10 Couchbase Webコンソール - サーバーノード


サーバー名をクリックすると、IPアドレス、OS、Couchbaseのバージョン、メモリとディスクの割り当て情報を含む、サーバノード固有の情報を提供します。

サーバをリストから選択するとサーバ固有のパフォーマンス情報、バケットの全ノードにわたる統計情報とともに、サーバ固有のバケット統計情報が表示されます。

図6.11 Couchbase Webコンソール、データバケット/サーバビュー


サーバ固有のグラフは以下のとおりです。

  • swap usage

    このサーバーで使用中のスワップ領域の量。

  • free memory

    このサーバー上で使用可能なRAMの量。

  • CPU utilization

    選択されたサーバー上のすべてのコアで使用されているCPUの割合。

  • connection count

    クライアント、プロキシ、TAPのリクエストおよび内部統計、すべてのタイプのこのサーバへの接続数。

プルダウン選択を使って、統計を閲覧する個々のバケットとサーバを選択できます。そしてミニグラフをクリックすると任意の統計情報を選択できます。

図6.12 Couchbase Webコンソール - サーバー固有のビュー


データバケットの統計の詳細については、「データバケット」 を参照してください。
