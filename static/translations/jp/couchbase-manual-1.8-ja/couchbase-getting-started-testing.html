2.5. Couchbase Serverのテスト

Telnetを使用してサーバーへ接続し、memcachedのテキストプロトコルを利用することで、Couchbase Serverのインストールをテストすることができます。これはCouchbase Serverが稼動しているかどうかを確認するための最も簡単な方法です。

注記

アプリケーション内から、サーバーと通信するためにTelnetを使用する必要はありません。代わりに、Couchbase SDKのいずれかを使用します。　

この手順ではCouchbase Serverに接続するためにtelnetのインストールが必要です。Telnetはほとんどのプラットフォーム上で標準で提供されています、もしくはオペレーティングシステム標準のパッケージマネージャから容易にインストール可能なパッケージとして提供されています。

サーバに接続します： 

shell> telnet localhost 11211
Trying 127.0.0.1...
Connected to localhost.localdomain (127.0.0.1).
Escape character is '^]'.

応答していることを確認してください（statsは基本的な状態チェックに便利です）： 

stats
STAT delete_misses 0
STAT rejected_conns 0
STAT rejected_conns 0
...
STAT time 1286678223

...
STAT curr_items_tot 0
...
STAT threads 4
STAT pid 23871
...
END

Couchbase Serverにドキュメントを保存します： 

set test_key 0 0 1
a
STORED

ドキュメントを取得します： 

get test_key
VALUE test_key 0 1
a
END

接続を解除します： 

{0} quit {/0}
Connection closed by foreign host.
shell>

memcachedプロトコルのコマンドはすべて、telnetから利用できます。
