6.3.3. 個々のバケットの監視

Couchbase Serverは、管理者が個々のノードとクラスタ全体の現在の状態をよりよく理解できるように、概要と詳細情報を表示するData BucketsとServer Nodesを通して利用可能な一連の統計情報とユーザインターフェイスを搭載しています。

Data Bucketsのページは、（MemBaseとmemcachedタイプの両方の）システム上の設定されたすべてのバケットのリストを表示します。このページでは、クラスタ全体または個々のサーバではなく、バケットの観点から、クラスタの動作状態を素早く概観することができます。

下の図に示すように、情報は、表形式で表示されます。

図6.2 Couchbase Webコンソール - データバケットの概要


各バケットについて、次の情報が各列に表示されます。

  • バケットタイプが表示アイコンで示されます。couchアイコンはMemBaseバケットを示します。Mアイコンは、memcachedバケットを示します。

  • Bucket nameは、バケットの名前です。バケット名をクリックすると、個々のバケットの統計ページが表示されます。詳細は、「個々のバケットの監視」を参照してください。

  • RAM Usage/Quotaは、設定されているバケットサイズに対して、（アクティブオブジェクトのために）使用されるRAMの量を示しています。

  • Disk Usageは、アクティブオブジェクトのデータ格納のために使用中のディスク領域の量を示しています。

  • Item Countは、バケットに格納されているオブジェクトの数を示します。

  • Ops/secは、このデータバケットの秒間オペレーション数を示します。

  • Disk Fetchs/secは、ディスクから項目をフェッチするのに必要な操作の数を示しています。

  • Informationボタンをクリックすると、基本的なバケット情報の要約を開きます。詳細は、「バケット情報」を参照してください。

新しいデータバケットを作成するには、 Create New Data Bucketをクリックします。 新しいデータバケットの作成については、「データバケットの作成と編集」を参照してください。
