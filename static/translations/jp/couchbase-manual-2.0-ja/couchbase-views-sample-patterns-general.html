9.9.1. 一般的な助言

あらゆるビューを記載するための一般的なポイントとアドバイスがあり、これはドキュメント構造、クエリの形式、もしくはビューの内容にかかわりなく適用されます。

  • フィールドがすべてのドキュメントに存在すると仮定しない

    フィールドがドキュメントには設定されていない場合や、特定のドキュメントタイプでのみ存在するかもしれません。問題を識別するためにifの条件式を使用してください。 例:

    Javascript
    if (document.firstname)...

  • ビューの出力は大文字と小文字が区別される

    emit()関数によって出力される値では大文字小文字を区別します。 'martin'のkeyの値を指定しているにも関わらず'Martin'のフィールド値を出力するというのはデータが一致していません。出力するデータ、およびキーの選択値は、潜在的な問題を排除するために正規化する必要があります。例:

    Javascript
    emit(doc.firstname.toLowerCase(),null);

  • 数値の形式

    JavaScript内の数値が誤って文字列として変換して出力されることがあります。 データが正しくフォーマットされていることを確認するには、値を明示的に変換する必要があります。例:

    Javascript
    emit(parseInt(doc.value,10),null);

    ビルトインのparseInt()関数は、指定された値を整数へ変換します。parseFloat()関数は、浮動小数点数に変換します。
