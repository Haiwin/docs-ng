4.3. デプロイの考慮事項

  • Moxiのポートへのアクセスを制限

    Moxiが利用する全てのポートが信頼できるマシン（クラスタ内のノードを含む）からのみアクセス可能としてください。

  • WEBコンソールへのアクセス制限 (ポート8091)

    Webコンソールはパスワードで保護されています。しかし、第三者がWebコンソールから悪意ある操作をする可能性があるため、ポート8091へのアクセスを制限することを推奨します。

  • ノード間の通信に使うポート

    クラスタ内のすべてのノードは、11210と8091で互いに通信できることが必要です。

  • スワップの構成

    システム用RAMが枯渇した際にオペレーティングシステムがCouchbase Serverをkillしないように、サーバ上のスワップを設定すべきです。スワップを利用すれば、このような状況を管理する上でより多くの選択肢を用意することができます。

  • アイドル状態の接続タイムアウト

    一部のファイアウォールまたはプロキシソフトウェアは、一定時間（例えば、20分）アイドル状態になっているTCP接続を切断します。ソフトウェアがそのタイムアウトを変更することが許可されていない場合、接続を維持するために定期的にクライアントからコマンドを送信します。

  • Windowsにおけるポートの枯渇

    WindowsでのデフォルトのTCP / IPポートの割り当ては、クライアント通信に使用可能なポートの制限値を含みます。設定を調整し、使用可能なポートを増やす方法についての情報を含むこの問題に関する詳細については、次の項目を}参照してください：MSDN: TCP / IPポートの枯渇の回避 。
