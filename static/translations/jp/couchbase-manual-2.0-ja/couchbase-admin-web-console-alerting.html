6.10. 警告とアラート

新しいアラートシステムがCouchbase Webコンソールに組み込まれています。これは、特定の問題と知っておくべき問題点を目立つようにと懇願されたためであり、また、Couchbaseクラスタの健全性を確保するために確認する必要があるためです。

アラートは、Webコンソール内でポップアップ表示されます。IPアドレスのポップアップのサンプルは以下のとおりです。

図6.35 Webコンソール — 警告通知


次のエラーとアラートがサポートされています。

  • IP Address Changes

    クラスタのCouchbase ServerのIPアドレスが変わった場合、そのアドレスは使用できなくなったと警告されます。サーバ上のIPアドレスをチェックし、クライアントまたはサーバの構成をアップデートする必要があります。

  • OOM (Hard)

    ノード上のバケットメモリが、完全にメタデータ用に使用されているかどうかを示します。

  • Commit Failure

    特定のバケットのデータをディスクに書き込むことに失敗したことを示します。

  • Metadata Overhead

    バケットは、データ用に使用可能なRAMの量を削減し、割り付けられたRAMの50％以上をメタデータとキーの格納に現在使用しているということを示します。

  • Disk Usage

    永続保存用に使用可能なディスク領域は、容量の少なくとも90％に達したことを示します。
