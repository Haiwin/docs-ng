6.3.1. データバケットの作成と編集

6.3.1.1. 新規バケットの作成
6.3.1.2. Couchbaseバケットの編集
6.3.1.3. Memcachedバケットの編集

新しいデータバケットを作成、または既存のものを編集するとき、バケットの設定画面が表示されます。ここからは、メモリのサイズ、アクセス制御、既存のバケットの編集か新規バケットの作成かに応じて、その他の設定やバケットタイプを設定することができます。

6.3.1.1. 新規バケットの作成

新しいバケットを作成するとき、次の図のようにバケット作成ダイアログが表示されます。

図6.12 Webコンソール — バケット作成


  • バケット名

    バケット名。バケット名は、下線、ピリオド、ダッシュおよびパーセント記号と、A-Z、aｰz、0-9の範囲の文字のみを含めることができます。

  • バケットタイプ

    作成するバケットのタイプをMemcachedかCouchbaseのどちらかで指定してください。詳細は「データストレージ」を参照してください。

  • アクセス制御

    アクセス制御は、クライアントがデータバケットとの通信に使用するポートと、そのバケットはパスワードが必要かどうかを設定します。

    TCP標準のポート（11211）を使用するため、作成する最初のバケットは、SASL認証を必要とせずに、このポートを使用することができます。それに続く各バケットでは、SASL認証に使用するパスワードを指定する必要があり、クライアントの通信は、バイナリプロトコルを使用して行う必要があります。

    専用のポートを使用するには、dedicate port ラジオボタンを選択し、使用するポート番号を入力します。専用のポートを使用すると、テキストとバイナリの両方のクライアントプロトコルをサポートし、認証を必要としません。

  • メモリサイズ

    このオプションは、このサーバ上で設定可能なRAMの量を指定し、構成しているバケットに割り当てます。割り当ては、すべてのノード間でのバケットの合計容量ではなく、各ノードでこのバケットに割り当てられるメモリ量であることに注意してください。

    警告

    memcachedのバケットのサイズを変更すると、バケット内のすべてのデータを消去して再作成するので、すべての保存されたデータが失われます。

  • レプリカ

    Couchbaseバケットは、レプリケーションにより、クラスタ内のサーバ間でバケットのレプリカを複数作成することができます。3つまでレプリカを設定できます。各レプリカは、バケットによって管理されているすべてのキー/値のペアのコピーを受け取ります。バケットをホストするマシンに障害が発生した場合でも、レプリカがその代わりとなることで、継続的な（高可用性）クラスタオペレーションを提供できます。

    レプリカコピーの数をゼロ（0）に設定することにより、レプリケーションを無効にすることができます。

    レプリカのインデックス（インデックス情報の新しいコピーが各ノードのレプリカデータから再作成されます）を構成するには、インデックスのレプリカ選択してください。

  • 自動コンパクション

    ディスクに保存されたデータとインデックスの両方が断片化します。コンパクションは、データの断片化を軽減するために、索引に格納されたデータを再構築します。データベースとビューのコンパクションの詳細については、「データベースとビューのコンパクション」を参照してください 。

    個々のバケットの構成でデフォルトの自動コンパクションの設定を上書きすることができます。デフォルト設定は、設定メニューを使用して構成されます。デフォルト自動コンパクションパラメータの設定の詳細については、「自動コンパクションの有効化」を参照してください 。デフォルトの自動コンパクション設定を上書きする場合は、同じパラメータを設定できますが、限度はこのバケットにのみ影響します。

  • フラッシュ

    Flushコマンドのサポートを有効または無効にします。Flushコマンドはバケット内の全データを削除します。デフォルトでは、flushコマンドは無効です。バケットへのフラッシュ操作を有効にするには、有効チェックボックスをクリックします。

バケット構成ウィンドウの下部にあるボタンでは、以下の操作をサポートしています。

  • 削除

    構成されたバケットを削除し、保存された関連コンテンツをすべて削除します。

  • フラッシュ

    フラッシュが有効になっている場合、フラッシュボタンをクリックすると、バケットとその構成を残して、構成済みバケットからすべてのデータを空にします。フラッシュ操作が行われる前に、操作を確認するメッセージが表示されます。

    注記

    フラッシュサポートが有効になっている場合のみ、フラッシュボタンが表示され、バケットの構成を開くことができます。フラッシュ設定を変更した場合、保存をクリックし、再度フラッシュ操作を実行するため、バケットの構成ダイアログを開く必要があります。

  • キャンセル

    バケットへの変更を取り消して、ウィンドウを閉じます。

  • 保存

    バケットへの変更を保存します。

6.3.1.2. Couchbaseバケットの編集

限られた範囲で既存のCouchbaseのバケットの設定を編集できます。

  • アクセス制御は、標準ポート/パスワード、もしくはカスタムポートの設定を含みます。

  • メモリサイズは、クラスタ内に未割り当て領域があれば、変更することができます。その領域がまだ使用されていない場合には、バケットに割り当てられるメモリの量を減らすことができます。

  • 自動コンパクション設定は、デフォルトの自動コンパクション設定の上書きを有効化および、バケット固有の自動コンパクションを設定できます。

  • フラッシュをサポートします。フラッシュコマンドのサポートを有効または無効にできます。

バケットの名前は変更できません。

構成済みバケットを完全に削除するには、削除ボタンをクリックしてください。

6.3.1.3. Memcachedバケットの編集

Memcachedバケットについては、既存のバケットに対し次の設定を変更することができます。

  • アクセス制御は、標準ポート/パスワード、もしくはカスタムポートの設定を含みます。

  • メモリサイズは、クラスタ構成内に未割り当てRAM領域があれば、変更することができます。その領域がまだ使用されていない場合には、バケットに割り当てられるメモリの量を減らすことができます。

削除ボタンをクリックして、バケットを完全に削除することができます。

フラッシュボタンを使用して、格納しているすべてのキャッシュ情報の入ったMemcachedバケットを空にすることができます。　

警告

フラッシュボタンを使用すると、Memcachedのバケットに格納されているすべてのオブジェクトを削除します。 アクティブなMemcachedバケットでこのボタンを使用すると、重要な情報を削除する場合があります。
